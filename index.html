Everything You Need

Production-ready features out of the box

🔐

Dual Authentication

Google OAuth one-click sign-in and Email OTP via Gmail SMTP. Both work out of the box.

💬

1-to-1 Chats

Search any registered user by name and start a direct conversation instantly with realtime delivery.

👥

Group Chats

Create named groups, add/remove members, admin controls, and a leave group option.

Realtime Updates

Firestore realtime listeners push new messages instantly. No polling needed — messages appear as they're sent.

Firestore security rules, OTP rate limiting, server-only env vars, and 10-minute OTP expiry.

Setup in 4 Steps

From zero to deployed in ~15 minutes

1

Create a Firebase Project

Go to console.firebase.google.com → New project → Enable Authentication (Google + Email/Password) → Create Firestore database.

5 minutes Free tier
2

Configure Environment Variables

Copy .env.local.example → .env.local and fill in Firebase config + Gmail credentials.

# Firebase keys
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
NEXT_PUBLIC_FIREBASE_PROJECT_ID=my-app
# Gmail OTP
GMAIL_USER=you@gmail.com
GMAIL_APP_PASSWORD=abcd efgh ijkl mnop
3

Get Gmail App Password Important

This is needed for Email OTP. It's NOT your regular Gmail password.

  1. Go to myaccount.google.com/security
  2. Enable 2-Step Verification
  3. Search for "App passwords"
  4. Select Mail → Other → type ChatApp → Generate
  5. Copy the 16-character code → paste in GMAIL_APP_PASSWORD
4

Deploy

▲ Vercel (Recommended)

Push to GitHub → Import on vercel.com → Add env vars → Deploy

npx vercel

◆ Netlify

Connect repo on netlify.com → netlify.toml auto-detected → Deploy

netlify deploy --prod

⚠️ After deploy: Add your domain to Firebase → Auth → Authorized domains
